2.  **Цель и назначение проекта**

**Цель:** Разработать веб-сервис для поиска и сравнения авиабилетов с
возможностью фильтрации по различным параметрам, аналогичный Aviasales,
но с уникальным дизайном и дополнительным функционалом.

**Задачи, решаемые проектом:**

-   Предоставление пользователям возможности поиска авиабилетов по
    любому направлению с актуальными (генерируемыми) ценами и условиями.

-   Сохранение понравившихся рейсов в избранное

-   Личный кабинет с возможностью загрузки аватара

-   История поисковых запросов для авторизованных пользователей

-   Предоставление пользователям возможности поиска авиабилетов по
    любому направлению с актуальными (генерируемыми) ценами и условиями.

2.  **Поиск и фильтрация (Ключевая особенность)**

    -   **Динамическая генерация рейсов:** В отличие от классического
        подхода, поиск не обращается к статической базе данных всех
        рейсов. Вместо этого, для выбранных пользователем городов и дат
        система генерирует новые рейсы на лету, используя информацию из
        базы данных аэропортов (коды, таймзоны) и авиакомпаний.

    -   **Страница результатов (**/results**):**

        -   Отображение найденных рейсов (туда и, при выборе, обратно).

        -   **Фильтрация:**

            -   По цене (диапазон).

            -   По количеству пересадок (0, 1, 2+).

            -   Только рейсы с включенным багажом.

            -   По времени вылета (утро, день, вечер, ночь).

        -   **Сортировка:** по возрастанию/убыванию цены, по времени
            вылета, по длительности.

        -   **Интерактивность:** При клике на карточку рейса открывается
            модальное окно с детальной информацией, загружаемой через
            REST API (/api/flight/\<int:flight_id\>).

        -   **Выбор рейсов:** Для маршрута \"туда-обратно\" реализован
            двухэтапный выбор: сначала пользователь выбирает билет
            \"туда\", после чего для него активируется выбор билета
            \"обратно\".

3.  **Карточка рейса и бронирование**

    -   **Страница деталей (**/flight/\<flight_id\>**):** Отображает
        полную информацию о рейсе: города, аэропорты, время
        вылета/прилета в локальном времени, длительность, информацию о
        пересадках (города и время ожидания), наличие багажа.

    -   **Форма бронирования:** На странице деталей пользователь может
        изменить класс обслуживания и количество пассажиров. Цена
        пересчитывается динамически с помощью JavaScript. Классы
        обслуживания имеют множители к базовой цене (Эконом 1.0, Комфорт
        1.5, Бизнес 2.5, Первый 4.0).

    -   **Бронирование:** После подтверждения создается запись в БД,
        генерируется уникальный номер бронирования, и пользователь
        перенаправляется в личный кабинет.

4.  **Регистрация и авторизация**

    -   Форма регистрации (имя пользователя, email, пароль) с валидацией
        данных и защитой от повторной регистрации.

    -   Хеширование паролей с использованием werkzeug.security.

    -   Защита от повторной регистрации

5.  **Личный кабинет (**/profile**)**

    -   Отображение информации о пользователе: имя, email, дата
        регистрации.

    -   **Загрузка аватара:** Пункт, закрывающий требование \"загрузка и
        использование файлов\". Пользователь может загрузить
        изображение, которое обрабатывается с помощью Pillow (изменение
        размера, оптимизация). Аватар отображается в навигационной
        панели и профиле. Используется url_for для корректной генерации
        ссылок на загруженные файлы.

    -   Вкладки:

        -   **Избранное:** Список рейсов, добавленных пользователем.
            Можно удалить из избранного.

        -   **История поиска:** Сохраняет последние 10 запросов
            пользователя с возможностью очистки истории.

        -   **Мои бронирования:** Отображает список всех бронирований
            (как в одну сторону, так и туда-обратно) с краткой
            информацией и статусом.

7.  **Бронирование туда-обратно**

    -   Реализована модель RoundTripBooking для хранения бронирований,
        включающих два рейса (туда и обратно).

    -   Пользовательский путь: поиск -\> выбор билета туда -\> выбор
        билета обратно -\> подтверждение -\> создание бронирования.

    1.  **Дополнительный функционал**

-   **Избранное** --- сохранение понравившихся рейсов в отдельный список

-   **История поиска** --- сохранение последних запросов пользователя

-   **Анимации** --- плавные переходы при наведении на карточки рейсов

-   **Адаптивный дизайн** --- корректное отображение на мобильных
    устройствах

-   Безопасная отмена бронирования: Для отмены бронирования пользователь
    должен пройти простую CAPTCHA (решить пример \"2+2\"), что является
    дополнительной мерой защиты от случайных действий.

**7. Тестовые данные**

Для демонстрации работы сервиса создаются следующие тестовые данные:

-   **Аэропорты:** 

База данных содержит 60+ аэропортов по всей России с указанием
IATA-кодов, названий, городов и часовых поясов.

-   **Генерация рейсов:** 

При первом запуске создается 400 статических рейсов для демонстрации на
главной странице. При поиске же рейсы **генерируются на лету** для
указанных дат и направлений, используя данные об аэропортах и
авиакомпаниях. Количество генерируемых рейсов варьируется от 8 до 15 на
направление.

-   **Авиакомпании:** 

10 компаний (Аэрофлот, S7, Уральские авиалинии и др.).

-   **Цены:** 

от 3 000 до 80 000 рублей (базовая цена).

-   **Пересадки:** 

Распределение вероятностей: 55% без пересадок, 30% с 1 пересадкой, 15% с
2 пересадками. Для рейсов с пересадками генерируется детальная
информация (город, аэропорт, время ожидания).
